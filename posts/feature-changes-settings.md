## Feature Changes — What We Added, Then Removed, Then Hid

The Little Wins iPhone app started as a win tracker. Simple. Then I kept adding things. Here's the full journey of what went in, what came out, and what's hidden.

### What We Added Then Removed

**Focus Mode (🎯)** — A single-focus-at-a-time screen. Set one task, complete it, move on. Sounded good for ADHD brains. In practice: it was another tab I never used. I track focus by just using Wins. Removed it entirely.

**ParNotes (📝)** — Quick-capture scratchpad. Like Apple Notes but inside the app. Same problem: another tab, another thing to maintain. Long-press to delete was the only interaction. Removed it.

Both had their own storage, their own screens, their own help docs. Cleaned all of it out. The app is now just **Wins** and **Settings**.

### What Settings Has

Settings started empty. Now it has:

- **Stats** — Total wins count. That's all.
- **Theme** — Dark / Light / System. Persisted.
- **Deen** — Toggle for the Astaghfirullah (☪) floating counter button. Tap to count, long-press to reset, drag to reposition.
- **How it works** — Tells you to tap ❔ in the header for per-screen help.
- **Test Data** — Add 20, 100, or 365 sample wins. For testing.
- **Clear All Data** — Nuke everything.

### What We Hid

The **Deen** and **Test Data** sections are hidden by default. Regular users don't need them. Tap the footer text ("Little Wins v1.0.0") 5 times to toggle **Developer Mode** — those sections appear. The footer text turns 🔧 to confirm.

The **Astaghfirullah button** is also hidden by default. Enable it in Settings → Deen (after unlocking dev mode).

### Help System

Every screen now has a ❔ icon in the header. Tap it → bottom sheet with screen-specific help cards. Wins screen has 6 cards explaining each feature. Settings has 6 cards covering theme, deen, stats, test data, privacy, keyboard shortcuts.

### Why This Keeps Happening

ADHD brain: "This feature would be useful!" → builds it → never uses it → removes it. The pattern repeats. The app gets smaller, not bigger. That's probably the right direction.

### Current State

- 3 onboarding slides (was 5)
- 2 screens: Wins + Settings (was 4 tabs)
- Settings accessible via ⚙️ icon in header (was a tab)
- Back from Settings via ✕ button (top-left)
